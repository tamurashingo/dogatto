# ラベル管理画面仕様

## 概要

ラベルの一覧表示、検索、作成、編集、削除、マージを行う管理画面。ラベルは複数のタグの組み合わせを表現し、TODO検索の効率化を図る。

## URL

- エンドポイント: `/labels`
- アクセス制御: ログイン済みユーザーのみ

## 画面構成

### レイアウト

```
+--------------------------------------------------+
| DOGATTO                    [ユーザーメニュー]     |
+--------------------------------------------------+
| ラベル管理                                        |
+--------------------------------------------------+
| 検索: (•) ラベル名 ( ) タグ名                     |
| [検索: ___________] [🔍]            [+ 新規作成]  |
+--------------------------------------------------+
| 全ラベル: 15件 | 使用中: 12件 | 未使用: 3件       |
+--------------------------------------------------+
| ラベル名 ▲  | タグ                | TODO数 | 操作 |
|------------|---------------------|--------|------|
| 朝活       | [MORNING🟠][ACTIVITY🔵] | 5件  | ... |
| 筋トレ     | [WORKOUT🟢][TUESDAY🟡]  | 8件  | ... |
| プロジェクトA | [PROJECT-A🟣][WORK⚫] | 3件  | ... |
| (未使用)   | [OLD-TAG⚪]          | 0件  | ... |
+--------------------------------------------------+
| ページネーション: < 1 2 3 >                      |
+--------------------------------------------------+
```

## 機能

### 1. ラベル一覧表示

**表示内容:**
- ラベル名
- 所属タグ（バッジ表示、タグの色付き）
- TODO数（そのラベルでヒットするTODO数）
- 最終更新日
- 操作ボタン（編集、削除）

**タグ表示:**
```
[MORNING 🟠] [ACTIVITY 🔵] [EXERCISE 🟢]
```
- タグ名 + タグの色を円で表示
- 最大5つまで表示、それ以上は「+2」のように表示
- ホバーで全タグを表示

**ソート:**
- ラベル名（昇順/降順）
- タグ数（昇順/降順）
- TODO数（昇順/降順）
- 更新日（昇順/降順）

**フィルタ:**
- すべてのラベル
- 使用中のラベル（TODO数 > 0）
- 未使用のラベル（TODO数 = 0）

**ページネーション:**
- 20件/ページ
- ページ番号表示
- 前へ/次へボタン

### 2. ラベル検索

**検索モード:**

#### モード1: ラベル名で検索（デフォルト）
- ラベル名での前方一致検索
- リアルタイム検索（入力中にフィルタ）
- 大文字小文字を区別しない

**検索UI:**
```
+--------------------------------------------------+
| 検索: (•) ラベル名 ( ) タグ名                     |
| [検索: 朝________________] [🔍] [×クリア]         |
+--------------------------------------------------+
| 検索結果: 2件                                    |
| 朝活    [MORNING🟠][ACTIVITY🔵]  5件            |
| 朝のルーチン [MORNING🟠][ROUTINE⚪] 3件          |
+--------------------------------------------------+
```

#### モード2: タグ名で検索
- タグ名での検索
- そのタグを含むラベルを検索
- 複数タグの場合はOR検索

**検索UI:**
```
+--------------------------------------------------+
| 検索: ( ) ラベル名 (•) タグ名                     |
| [検索: MORNING________] [🔍] [×クリア]           |
+--------------------------------------------------+
| 検索結果: 3件                                    |
| 朝活    [MORNING🟠][ACTIVITY🔵]  5件            |
| 朝のルーチン [MORNING🟠][ROUTINE⚪] 3件          |
| モーニングワーク [MORNING🟠][WORK⚫] 2件         |
+--------------------------------------------------+
```

**検索実装:**
- Debounce: 300ms
- 最小文字数: 1文字
- 検索結果のハイライト表示

**検索切り替え:**
- ラジオボタンで検索モード切り替え
- 検索モード変更時に自動的に再検索

### 3. ラベル新規登録

**トリガー:**
- 「+ 新規作成」ボタンをクリック

**表示形式:**
- モーダルダイアログ

**入力項目:**
```
+--------------------------------------------------+
| ラベルの作成                              [×]     |
+--------------------------------------------------+
| ラベル名 *                                       |
| [_____________________________]                  |
|                                                  |
| 説明                                            |
| [_____________________________]                  |
|                                                  |
| タグを追加 *                                     |
| [タグを選択 ▼]                                   |
|                                                  |
| 選択中のタグ:                                    |
| [MORNING 🟠 ×] [ACTIVITY 🔵 ×]                  |
|                                                  |
| プレビュー:                                      |
| ラベル「朝活」でTODOを検索すると、                |
| タグ「MORNING」AND「ACTIVITY」を持つTODOが       |
| 表示されます。                                   |
|                                                  |
| 推定TODO数: 5件                                  |
|                                                  |
|                         [キャンセル] [作成]       |
+--------------------------------------------------+
```

**バリデーション:**
- ラベル名: 必須、1-100文字
- タグ: 最低1つ必須
- ラベル名: 重複チェック（既存ラベルと比較）

**タグ選択ドロップダウン:**
```
┌─────────────────────────────┐
│ [検索: _____________]       │
├─────────────────────────────┤
│ MORNING  🟠  8件のTODO      │
│ ACTIVITY 🔵  12件のTODO     │
│ WORKOUT  🟢  15件のTODO     │
│ DAILY    🟡  20件のTODO     │
└─────────────────────────────┘
```

**ドロップダウン仕様:**
- 既存タグから選択
- タグ名で検索可能
- タグの色と使用数を表示
- 既に選択済みのタグは無効化
- キーボードナビゲーション対応（↑↓、Enter）

**動作:**
1. ラベル名を入力
2. 説明を入力（オプション）
3. タグを選択して追加
4. 推定TODO数を表示（リアルタイム計算）
5. 「作成」ボタンをクリック
6. API呼び出し
7. 成功時: モーダルを閉じ、一覧に追加、トースト通知
8. 失敗時: エラーメッセージ表示

**推定TODO数の計算:**
- 選択したタグの組み合わせでヒットするTODO数をリアルタイム計算
- API: `GET /api/labels/estimate-todo-count?tag_ulids=xxx,yyy`

**トリガー:**
- 操作メニュー（...）→「編集」をクリック
- ラベル行をダブルクリック

**表示形式:**
- モーダルダイアログ

**入力項目:**
```
+--------------------------------------------------+
| ラベルの編集                              [×]     |
+--------------------------------------------------+
| ラベル名 *                                       |
| [朝活_____________________]                      |
|                                                  |
| 説明                                            |
| [朝の活動タスク____________]                      |
|                                                  |
| タグを追加                                       |
| [タグを選択 ▼]                                   |
|                                                  |
| 選択中のタグ:                                    |
| [MORNING 🟠 ×] [ACTIVITY 🔵 ×] [EXERCISE 🟢 ×] |
|                                                  |
| 使用状況:                                        |
| このラベルでヒットするTODO: 5件                  |
|                                                  |
| ⚠ タグを変更すると、検索結果が変わります          |
|                                                  |
|                         [キャンセル] [更新]       |
+--------------------------------------------------+
```

**バリデーション:**
- ラベル名変更時の重複チェック
- タグは最低1つ必須

**確認ダイアログ（タグ変更時）:**
```
+--------------------------------------------------+
| ラベルのタグを変更しますか？                      |
+--------------------------------------------------+
| ラベル「朝活」のタグを変更します。                |
|                                                  |
| 変更前のタグ: MORNING, ACTIVITY                 |
| 変更後のタグ: MORNING, ACTIVITY, EXERCISE       |
|                                                  |
| 変更前のTODO数: 5件                             |
| 変更後のTODO数: 8件                             |
|                                                  |
| ⚠ この変更により検索結果が変わります              |
|                                                  |
|                         [キャンセル] [変更]       |
+--------------------------------------------------+
```

**動作:**
1. ラベル情報を取得してフォームに表示
2. ラベル名、説明、タグを変更
3. 「更新」ボタンをクリック
4. タグ変更の場合は確認ダイアログ表示
5. API呼び出し
6. 成功時: モーダルを閉じ、一覧を更新、トースト通知
7. 失敗時: エラーメッセージ表示

**タグの追加:**
- ドロップダウンからタグを選択
- 選択したタグが「選択中のタグ」に追加

**タグの削除:**
- タグバッジの「×」をクリック
- 最後の1つは削除不可（最低1つ必須）

### 5. ラベルマージ

**トリガー:**
- 画面上部の「マージ」ボタンをクリック
- または操作メニュー（...）→「マージ」をクリック

**表示形式:**
- 専用のマージ画面へ遷移（`/labels/merge`）
- または大きなモーダルダイアログ

**マージUI（簡易版）:**
```
+--------------------------------------------------+
| ラベルのマージ                            [×]     |
+--------------------------------------------------+
| マージ元ラベルを選択:                            |
| [朝かつ ▼]                                       |
| タグ: [MORNING🟠][ACTIVITY🔵]                   |
|                                                  |
| マージ先を選択:                                  |
| ( ) 既存のラベルにマージ                          |
|     [朝活 ▼]                                     |
|     タグ: [MORNING🟠][ACTIVITY🔵][EXERCISE🟢]   |
| (•) 新しいラベルを作成                            |
|     ラベル名: [NEW-LABEL___________]             |
|     タグ: [MORNING🟠][ACTIVITY🔵]               |
|                                                  |
| 影響を受けるTODO:                                |
| - 朝かつ: 5件                                    |
| - 統合後: 8件                                    |
|                                                  |
|                         [キャンセル] [マージ]     |
+--------------------------------------------------+
```

**詳細:**
- ラベルマージ仕様（`label-merge-spec.md`）を参照

### 6. ラベル削除

**トリガー:**
- 操作メニュー（...）→「削除」をクリック

**確認ダイアログ:**
```
+--------------------------------------------------+
| ラベルを削除しますか？                            |
+--------------------------------------------------+
| ラベル「朝活」を削除します。                      |
|                                                  |
| このラベルでヒットするTODO: 5件                  |
|                                                  |
| ⚠ 削除してもTODOは削除されません                 |
| TODOのタグはそのまま残ります                     |
|                                                  |
| この操作は取り消せません。                        |
| 代わりにマージを検討してください。                |
|                                                  |
|                         [キャンセル] [削除]       |
+--------------------------------------------------+
```

**削除種類:**

1. **未使用ラベルの削除（論理削除）**
   - TODO数 = 0 のラベル
   - `deleted_at` に日時を設定
   - 確認ダイアログは簡易版

2. **使用中ラベルの削除（警告付き）**
   - TODO数 > 0 のラベル
   - 影響範囲を表示
   - マージ推奨メッセージ

**動作:**
1. 削除ボタンをクリック
2. 使用状況を取得
3. 確認ダイアログ表示
4. 「削除」ボタンをクリック
5. API呼び出し
6. 成功時:
   - 一覧から削除
   - トースト通知: "ラベルを削除しました"
7. 失敗時:
   - エラーダイアログ表示

**注意:**
- ラベルを削除してもTODOは削除されない
- TODOのタグはそのまま残る
- ラベルは検索条件のショートカットに過ぎない

## API仕様

### 1. ラベル一覧取得

**エンドポイント:** `GET /api/labels`

**クエリパラメータ:**
- `page`: ページ番号（デフォルト: 1）
- `per_page`: 1ページあたりの件数（デフォルト: 20、最大: 100）
- `sort`: ソート項目（name, tag_count, todo_count, updated_at）
- `order`: ソート順（asc, desc）
- `filter`: フィルタ（all, used, unused）
- `search_mode`: 検索モード（label_name, tag_name）
- `q`: 検索キーワード

**レスポンス:**

## バリデーション

### ラベル名

**クライアント側:**
- 必須チェック
- 文字数チェック（1-100文字）

**サーバー側:**
- 必須チェック
- 文字数チェック
- 重複チェック（ユーザー内）

**エラーメッセージ:**
- "ラベル名を入力してください"
- "ラベル名は100文字以内で入力してください"
- "このラベルは既に存在します"

### タグ

**必須チェック:**
- 最低1つのタグが選択されていること

**エラーメッセージ:**
- "タグを少なくとも1つ選択してください"

### 説明

**オプション:**
- 最大1000文字

## UI/UX要件

### 1. レスポンシブデザイン

**デスクトップ（1024px以上）:**
- テーブル表示
- すべてのカラムを表示
- タグは最大5つまで表示、それ以上は「+N」
- 操作メニュー: ドロップダウン

**タブレット（768px - 1023px）:**
- テーブル表示
- 一部カラムを非表示（更新日など）
- タグは最大3つまで表示
- 操作メニュー: ドロップダウン

**モバイル（767px以下）:**
- カード表示
```
┌─────────────────────────────┐
│ 朝活                        │
│ [MORNING🟠][ACTIVITY🔵]     │
│ 5件のTODO                   │
│ 更新: 1/10                  │
│ [編集] [削除]               │
└─────────────────────────────┘
```

### 2. アクセシビリティ

**キーボード操作:**
- `Tab`: 次の要素へ移動
- `Shift + Tab`: 前の要素へ移動
- `Enter`: ボタンのクリック、リンクの遷移
- `Space`: ラジオボタンのトグル
- `↑↓`: リスト内の移動、ドロップダウン内の移動
- `e`: 選択中のラベルを編集
- `Delete`: 選択中のラベルを削除

**ARIA属性:**
- `role="table"`: テーブル
- `role="button"`: ボタン
- `role="combobox"`: ドロップダウン
- `aria-label`: ボタンのラベル
- `aria-sort`: ソート状態
- `aria-live`: 動的な更新通知
- `aria-expanded`: ドロップダウンの開閉状態

**スクリーンリーダー対応:**
- ラベル名、タグ、TODO数、操作の読み上げ
- 検索結果の件数読み上げ
- エラーメッセージの読み上げ

### 3. パフォーマンス

**検索:**
- Debounce: 300ms
- キャンセル処理（前回のリクエストをキャンセル）

**一覧表示:**
- ページネーション（20件/ページ）

**TODO数推定:**
- Debounce: 500ms
- キャッシュ（同じタグの組み合わせ）

### 4. アニメーション

**ラベル追加:**
- リストの先頭に追加
- フェードイン + スライドダウン（300ms）

**ラベル削除:**
- フェードアウト + スライドアップ（300ms）
- リストの高さを調整

**モーダル:**
- 背景: フェードイン（200ms）
- コンテンツ: スライドアップ（300ms）

**タグバッジ追加/削除:**
- フェードイン/フェードアウト（200ms）

- すべてのPOST/PATCH/DELETE にCSRFトークン

### XSS対策

- ラベル名のエスケープ
- 説明のエスケープ

### レート制限

- ラベル作成: 10回/分
- ラベル更新: 20回/分
- ラベル削除: 5回/分
- TODO数推定: 30回/分
